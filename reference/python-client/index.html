    Python Client¶

    Installation¶

    pip install "quiver-python @ git+https://github.com/Olamyy/quiver.git#subdirectory=quiver-python"

    Basic usage¶

    from quiver import Client

client = Client("localhost:8815")

table = client.get_features(
    feature_view="user_features",
    entities=["user:1000", "user:1001"],
    features=["score", "country"],
)

# Convert to your preferred format
df = table.to_pandas()
arr = table.to_numpy()
tensor = table.to_torch()

    Client options¶

    client = Client(
    address="localhost:8815",
    timeout=30.0,           # request timeout in seconds
    max_retries=3,          # retry attempts on transient failures
    compression="gzip",     # optional: "gzip" or "zstd"
    validate_connection=True,  # ping server on init
)

    Temporal queries¶

    from datetime import datetime, timezone

table = client.get_features(
    feature_view="user_features",
    entities=["user:1000"],
    features=["score"],
    as_of=datetime(2024, 6, 1, tzinfo=timezone.utc),
)

    Observability¶

    Each request exposes a request ID for latency introspection:

    table = client.get_features(...)

request_id = client.get_last_request_id()
from_cache = client.get_last_from_cache()

metrics = client.get_metrics(request_id)
print(f"Total: {metrics['total_ms']:.1f}ms")
print(f"Merge: {metrics['merge_ms']:.1f}ms")
print(f"Redis: {metrics['backend_redis_ms']:.1f}ms")

    Context manager¶

    with Client("localhost:8815") as client:
    table = client.get_features(...)

    API Reference¶

    quiver.client.Client ¶

    Quiver client for feature serving with Arrow Flight integration.

    Provides strongly-typed interface to Quiver server with immediate validation. All network errors, configuration errors, and type errors are caught early.

    Parameters:

    Name Type Description Default
    address str

    Server address (e.g. "localhost:8815" or "grpc://localhost:8815")

    		 required
    timeout float

    Request timeout in seconds

    		 30.0
    max_retries int

    Number of retry attempts for failed requests

    		 3
    compression Optional[CompressionType]

    Optional compression algorithm

    		 None
    default_null_strategy NullStrategy

    How to handle null/missing values by default

    		 'fill_null'
    validate_connection bool

    Whether to validate connection on init

    		 False

    Raises:

    Type Description
    QuiverConnectionError

    If server is unreachable
    QuiverValidationError

    If configuration is invalid

    flush_cache() ¶

    Flush the server cache.

    This operation clears any cached feature data on the server, forcing fresh data retrieval on subsequent requests.

    Returns:

    Name Type Description
    bool bool

    True if cache was successfully flushed

    Raises:

    Type Description
    QuiverValidationError

    If client is closed
    QuiverServerError

    If server error occurs
    Note

    This is currently a no-op operation on the server side. Future implementations may add actual cache clearing functionality.

    get_features(feature_view, entities, features, *, as_of=None, null_strategy=None, include_timestamps=False, include_freshness=False, context=None, timeout=None) ¶

    Get features for specified entities.

    Parameters:

    Name Type Description Default
    feature_view FeatureViewName

    Name of the feature view to query

    		 required
    entities List[EntityId]

    List of entity IDs to get features for

    		 required
    features List[FeatureName]

    List of feature names to retrieve

    		 required
    as_of Optional[datetime]

    Point-in-time for temporal queries

    		 None
    null_strategy Optional[NullStrategy]

    How to handle null/missing values

    		 None
    include_timestamps bool

    Whether to include feature timestamps

    		 False
    include_freshness bool

    Whether to include freshness metadata

    		 False
    context Optional[RequestContext]

    Request context for tracing and debugging. Can include a custom request_id for correlation with other services.

    		 None
    timeout Optional[float]

    Request timeout override

    		 None

    Returns:

    Name Type Description
    FeatureTable FeatureTable

    Table with requested features

    Raises:

    Type Description
    QuiverValidationError

    If inputs are invalid
    QuiverFeatureViewNotFound

    If feature view doesn't exist
    QuiverFeatureNotFound

    If features don't exist
    QuiverConnectionError

    If connection fails
    QuiverServerError

    If server error occurs
    QuiverTimeoutError

    If request times out

    Pass a custom request_id for cross-service tracing: from quiver.models import RequestContext ctx = RequestContext(request_id="my-trace-id-12345") table = client.get_features( "user_profile", ["user:1000"], ["score"], context=ctx ) # Server will use "my-trace-id-12345" instead of generating UUID

    get_last_request_id() ¶

    Get the request ID from the last feature request.

    The request ID can be used to retrieve detailed metrics about the request from the observability service via get_metrics().

    Returns:

    Type Description
    Optional[str]

    The request ID if available, None if no request has been made yet
    Example

    table = client.get_features(...) request_id = client.get_last_request_id() if request_id: metrics = client.get_metrics(request_id) print(f"Total latency: {metrics['total_ms']:.2f}ms")

    get_metrics(request_id=None) ¶

    Retrieve detailed metrics for a feature request.

    Queries the observability service for latency metrics collected during feature resolution. Metrics include 11 instrumentation points covering registry lookup, dispatch, backend execution, merge, and more.

    Parameters:

    Name Type Description Default
    request_id Optional[str]

    The request ID to retrieve metrics for. If None, uses the request ID from the last feature request.

    		 None

    Returns:

    Type Description
    Dict[str, Any]

    Dictionary with latency metrics (all in milliseconds):
    Dict[str, Any]
    • registry_lookup_ms: Time to lookup feature view in registry
    Dict[str, Any]
    • cache_lookup_ms: Time to check metrics cache
    Dict[str, Any]
    • partition_ms: Time to partition features by backend
    Dict[str, Any]
    • dispatch_ms: Time to submit requests to backends
    Dict[str, Any]
    • backend_redis_ms: Redis backend latency (0.0 if not used)
    Dict[str, Any]
    • backend_delta_ms: Delta backend latency (0.0 if not used)
    Dict[str, Any]
    • backend_postgres_ms: PostgreSQL backend latency (0.0 if not used)
    Dict[str, Any]
    • backend_max_ms: Maximum latency across all backends
    Dict[str, Any]
    • alignment_ms: Time to align entity results
    Dict[str, Any]
    • merge_ms: Time to merge backend results via Arrow LEFT JOIN
    Dict[str, Any]
    • validation_ms: Time to validate merged results
    Dict[str, Any]
    • serialization_ms: Time to convert to Arrow IPC format
    Dict[str, Any]
    • total_ms: Total request latency
    Dict[str, Any]
    • critical_path_ms: Critical path latency
    Dict[str, Any]
    • feature_view: Feature view that was queried
    Dict[str, Any]
    • entity_count: Number of entities requested
    Dict[str, Any]
    • request_timestamp: When the request was processed

    Raises:

    Type Description
    QuiverValidationError

    If client is closed or no request_id available
    QuiverConnectionError

    If observability service unavailable
    QuiverServerError

    If request_id not found (metrics expired or invalid)
    QuiverTimeoutError

    If observability service times out
    Example

    table = client.get_features("user_features", ["user1", "user2"], ["score"]) metrics = client.get_metrics() # Uses last request ID print(f"Merge latency: {metrics['merge_ms']:.2f}ms")

    reload_registry() ¶

    Reload the feature registry.

    This operation forces the server to reload its feature view definitions from the configuration, allowing for dynamic configuration updates without server restart.

    Returns:

    Name Type Description
    bool bool

    True if registry was successfully reloaded

    Raises:

    Type Description
    QuiverValidationError

    If client is closed
    QuiverServerError

    If server error occurs
    Note

    This is currently a no-op operation on the server side. Future implementations may add actual registry reloading functionality.
